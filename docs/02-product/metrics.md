# 指标定义

> 状态：现行 ｜ 最后更新：2026-07-05 ｜ 原则：指标必须可测、刷不动、指向真实价值

## 北极星指标（NSM）

**每周成功交付且 7 日内仍被真实使用的应用数（Weekly Delivered & Retained Apps, WDRA）**

| 要素 | 口径 |
|---|---|
| 成功交付 | 通过[内测交付验收单](acceptance-test-scenarios.md#4-内测交付验收单每个真实客户项目)全部条目 |
| 7 日内仍被真实使用 | 交付后第 7 日，该应用 PocketBase 实例中存在**交付日之后创建的业务记录**（`app_data` 按 created 判断），且登录用户 ≥2 人 |
| 测量方式 | P1：人工回访 + 实例数据检查；P3：平台埋点自动化 |

**为什么是它**：同时度量生成质量（做得出）、交付闭环（交得掉）、真实价值（用得住）；比"周活创作者"更难刷、更贴近交付生意（ADR-007）。

## P1 阶段指标（当前唯一需要盯的一组）

| 指标 | 定义 | 目标 | 数据来源 |
|---|---|---|---|
| 首次生成成功率 | 稳定性实验口径：无人工干预产出 [BUILD_DONE] 且质量分 ≥3 | ≥80% | 实验记录 |
| 修改请求成功率 | 修改生效且不破坏原功能/数据 | ≥70% | 实验记录 |
| `c_create` | 单次创作会话平均 LLM 成本（元） | 实测回填 | agent-server token 统计 / DeepSeek 控制台对账 |
| 单次制作耗时 | 确认风格 → BUILD_DONE | ≤5 分钟 | 实验记录 |
| 内测交付数 | 通过交付验收单的真实客户项目 | ≥5（P1 退出） | 交付归档 |
| 交付周期 | 首次对话 → 客户验收通过 | 观察值（定价依据） | 交付归档 |

## P2+ 预备指标（定义好，先不考核）

| 指标 | 定义 | 用途 |
|---|---|---|
| 支付转化率 | 报价客户中付费比例（按 299/999/2999 档分层） | 定价决策 |
| 托管续费率 | 月度订阅续费 | 交付质量的滞后信号 |
| 毛利/项目 | 交付费 −（c_create × 会话数 + 人工兜底工时成本） | 单位经济闭环验证 |
| 人工介入率 | 交付项目中需要人工改代码的比例 | 产品化程度；目标随版本下降 |
| 案例墙转化 | 案例页访问 → 咨询留资 | P2 获客漏斗 |

## 反指标（不许用来汇报）

- ❌ 注册数 / 对话轮数 / 生成次数（无交付质量含义，易刷）
- ❌ 体验模式使用量（演示流量，只作获客漏斗分母）
- ❌ 代码行数 / 提交数
