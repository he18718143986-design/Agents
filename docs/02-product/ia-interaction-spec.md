# 信息架构与交互规范

> 状态：现行 ｜ 最后更新：2026-07-05 ｜ 术语表是文案与代码命名的唯一权威来源

## 1. 站点结构

```
/            营销首页（LandingPage：定位、对比、用例、作品展示、定价占位、CTA）
/app         应用
 ├─ 工作台（HomePage：继续创作卡片 + 全部项目列表 + 新建项目）
 └─ 工作区（AppShell：顶栏 + 左侧对话栏 + 右侧画布，可拖拽分栏）
生成应用     /api/conversations/<id>/workspace/…（agent-server 静态托管）
数据接口     /pb/<slug>/…（每应用 PocketBase 代理）
```

## 2. 工作区布局

| 区域 | 内容 | 组件 |
|---|---|---|
| 顶栏 | 返回工作台、项目名、引擎状态（体验模式/AI 已连接/未连接）、菜单（连接 AI/回到需求/回到风格/重新开始） | `WorkspaceTopBar` |
| 左栏（对话） | 消息流、快捷回复、**失败手册**（出错时）、**阶段门**（拍板按钮区）、输入框 | `LeftSidebar` = `ChatPanel` + `FailurePlaybook` + `GateBar` |
| 右栏（画布） | 随阶段切换：探索/需求/选型/预览/发布 五种画布，画布内用页签组织 | `CanvasPanel` + 各 Canvas + `CanvasTabs` |

**核心交互原则**：左边"聊"，右边"看和点"；所有不可逆推进必须经过阶段门按钮，聊天不能触发越级跳转。

## 3. 阶段机（唯一状态权威：`store.ts`）

```
阶段0 探索 ──path门── 阶段1 需求整理 ──requirements门── 阶段2 选型风格
   │（选 SaaS/低代码 → 外部方案引导页，流程结束）
阶段2 ──style门── 阶段3 制作与验收 ──acceptance门── 阶段4 测试与上线 ──go_live门── 完成
回退：任何阶段可回到更早阶段（顶栏菜单/门禁次按钮）；完成后可进入迭代模式（回到阶段1）
```

门禁显示条件 = **画布字段齐全 AND AI 判断就绪（gateHints）** 双条件（`gateReadiness.ts`）。

## 5. 文案原则

1. **诚实优先**：演示就说演示，真实保存就说真实保存；能力边界主动告知（`buildCapability.ts` 是能力文案唯一来源）
2. **人话**：面向王校长写作；技术细节收进"技术详情"折叠区
3. **错误三件套**：一句人话结论 + 可执行的下一步按钮 + 可折叠的技术详情（`FailurePlaybook` 模式）
4. 中文全角标点；界面不出现英文术语（Stagent 品牌名除外）

## 6. 生成应用（黄金模板）的 IA

```
登录屏（含演示账号提示）→ 应用壳（头部：应用名+数据徽标+用户+退出）
 → 模块页签（每 P0 功能一个）→ 模块面板 = 汇总卡片区 + 录入表单 + 记录表格（预警行标红）+ 导出按钮
```

约束：模板 `base.css` 定义结构样式，`custom.css` 只改 CSS 变量（品牌一致性）；预警行、汇总卡片的视觉语义全局统一。
