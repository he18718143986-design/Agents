# PRD：P1 内测版（范围冻结）

> 状态：**冻结** ｜ 最后更新：2026-07-05 ｜ 变更需走 ADR 并更新本文档
>
> 冻结含义：P1 期间只修 bug 和稳定性问题，不加新功能面；新想法记入 [roadmap](roadmap.md) 对应阶段。

## 1. 产品概述

Stagent P1 = **对话式应用生成工具（内测版）**：非技术用户通过五阶段拍板流程，把业务需求变成一个真实可用（数据云端保存、可登录、多人用）的网页应用。

两种运行模式：
- **体验模式**：内置脚本驱动全流程，无需密钥，用于演示/获客/静态托管环境
- **真实引擎**：OpenHands agent-server + DeepSeek，生成真实应用

## 2. 用户与场景

主用户：中小机构负责人（无技术背景）。场景：为自己团队（3–10 人）做管理工具（台账/预约/进度/课时类）。内测期由平台方陪跑操作。

## 3. 功能范围（In Scope）

### 3.1 五阶段流程（含阶段门）

| 阶段 | 功能 | 验收标准 |
|---|---|---|
| 0 探索 | 对话澄清想法；右侧需求草稿实时同步；SaaS/低代码/自研三方案对比；**允许以"买现成"结束** | 需求草稿含目标+P0 功能；门禁需"字段齐全 + AI 判断就绪"双条件才可拍板 |
| 1 需求整理 | 补全目标用户/验收标准/时间预期；可行性告知（第三方集成 mock 门禁） | 五要素齐全才可确认；涉第三方集成必须用户显式确认 mock 方案 |
| 2 选型与风格 | 技术路线（网页/小程序模拟/桌面模拟）+ 界面风格 A/B + 页面预览 | 两项都选才可开始制作 |
| 3 制作与验收 | 制作 Agent 基于黄金模板生成应用；试用预览；**验收清单由用户验收标准动态生成**；"还不行，继续改"→ 下一条消息作为修改要求提交 | 预览可实际操作；清单全勾选才可通过；修改请求生效且不破坏原功能 |
| 4 测试与上线 | 测试环境部署 → 上线检查清单 → 正式环境发布 | staging/production 目录落盘且可访问 |

### 3.2 生成应用的能力（baas-mvp 档位）

- 登录/注册（PocketBase users）、演示账号自动开通
- 数据云端保存（每应用独立 PocketBase 实例 + SQLite 落盘），刷新/换设备不丢
- 模块化台账：表单、表格、汇总卡片（count/sum/avg）、预警标红（lt/gt 规则）、CSV 导出
- 制作 Agent 只编辑声明式 `modules.js` + `custom.css`（黄金模板保证质量下限）

### 3.3 平台能力

- 引擎连接：云端密钥一键连接（环境变量探测）/ 手动填 key / 体验模式
- 项目工作台：多项目、快照恢复（localStorage，P1 已知限制）
- 生产部署形态：Node 生产服务器 + docker compose + nginx 配置（deploy/）
- 大陆网络适配：无境外 CDN 依赖、镜像化构建

## 4. 明确不在范围（Out of Scope，P1 不做）

- 平台账号体系、支付/计费、用量限额（P3）
- 每任务独立沙箱、生成应用独立子域名（P3）
- 验收自动化测试报告（P3）
- 微信小程序真机产出、原生 App（P4/远期）
- 生成应用的复杂流程引擎（多级审批/自动化任务）
- 移动端深度适配（可用即可，不做专项优化）

## 5. 非功能要求

| 项 | 要求 |
|---|---|
| 生成稳定性 | 首次成功率 ≥80%，修改成功率 ≥70%（稳定性实验口径） |
| 生成耗时 | 单次制作 ≤5 分钟（DeepSeek 实测约 3 分钟） |
| 数据安全 | 业务数据 SQLite 落盘 + 每日备份（部署文档含命令）；密钥不入库不入仓 |
| 访问控制 | 内测期 nginx Basic Auth；agent-server 端口不对公网 |
| 合规 | 备案通过前不绑域名；生成内容含"演示/真实数据版"如实标注 |

## 6. 已知限制（如实告知内测用户）

- 项目快照存浏览器本地，换设备不可恢复（P3 解决）
- 并发制作可能互扰（内测期串行安排客户，P3 解决）
- 第三方集成（微信/支付/短信）以 mock 演示
- 修改请求粒度：模块/字段/规则/配色级别；超出模板引擎能力的深度定制需人工介入
