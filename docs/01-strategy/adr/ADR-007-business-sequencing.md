# ADR-007：商业排序——交付生意（A）先行，生态生意（B）后置

> 状态：已采纳 ｜ 2026-07-05

## 背景

商业战略文档同时描绘了两条轨道：A）面向中小商户的软件交付工具；B）创作者生态平台（作品广场、验证收入榜、买卖撮合、广告）。两者客户画像、应用形态、收入结构、启动条件均不同。

## 决策

**A 养 B，串行推进**：P1–P3 以交付生意为主轴（交付费 + 托管订阅，平台持钥），P4 才启动生态层（广场变现、撮合、BYO 漏斗、可选广告）。P2 的"案例墙"是 B 的种子，但定位为营销资产。

## 备选与否决理由

| 备选 | 否决理由 |
|---|---|
| 直接做生态平台（B 先行） | 双边市场冷启动无现金流支撑；广告 ARPU 与算力成本差 1–2 个数量级；"验证收入榜"需要平台上先有赚钱的应用 |
| A/B 并行 | 团队规模不支持两条产品线；客户画像分裂导致产品决策摇摆（如 BYO key 对老板人群是负体验） |
| 只做 A（放弃 B） | B 的护城河（供给存量、原生验证收入、行业数据）是长期壁垒，且 A 的产出天然为其积累原料，无需现在放弃 |

## 后果

- 正：现金流前置；北极星指标聚焦（交付且被使用的应用数）；BYO/广告等复杂机制推迟到有承载能力时
- 负：生态先发优势让渡给竞品的风险；缓解：案例墙从 P2 就开始积累公开供给，托管+收款架构从 day one 保留"原生验证收入"的数据基础
- 触发重估条件：若 P2 发现交付生意获客成本不可控，或出现直接竞品以生态打法快速起量，重开本决策
