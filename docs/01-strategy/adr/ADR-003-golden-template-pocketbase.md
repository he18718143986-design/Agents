# ADR-003：L3 数据底座用 PocketBase 黄金模板（每应用一实例）

> 状态：已采纳 ｜ 2026-07-05（PR #6 实现）

## 背景

L3 要求生成的应用"数据真实保存、有登录、多人用"。让 LLM 从零写认证和数据库 = 每次都赌它不出安全漏洞。

## 决策

**底座由人工打磨的模板保证，AI 只填业务逻辑。** 采用 PocketBase（MIT，单二进制 SQLite）：每个应用一个独立实例（`server/pocketbase.ts` 管理，`/pb/<slug>/` 代理），模板（`prototype/templates/baas-static/`）内置登录、CRUD、汇总、预警、导出。

## 备选与否决理由

| 备选 | 否决理由 |
|---|---|
| Supabase（自托管） | Postgres 集群运维重，P1 规模不需要；保留为 P3+ 规模化升级方向 |
| NocoBase（AI 配置无代码平台） | 定制自由度低；保留为复杂权限类行业模板备选 |
| 共享单实例 + 集合前缀 | 租户隔离弱、备份/迁移/删除纠缠；每实例 SQLite 单文件备份是运维优势 |
| LLM 自由生成后端代码 | 安全与质量不可控，违背"质量下限由模板保证"原则 |

## 后果

- 正：数据隔离天然按应用；备份 = 拷文件；一台 ECS 可托管数百实例；无境外 CDN 依赖（SDK 内置）
- 负：每实例一个进程（数百级之后需要进程管理优化）；数据模型 v1 用 `app_data` 单集合 + JSON 字段，复杂查询能力有限（真实集合建模留待 P3）
- 已知坑（已修复并记录）：PB v0.23+ 时间戳字段需显式创建；JS SDK 并发请求需独立 requestKey
