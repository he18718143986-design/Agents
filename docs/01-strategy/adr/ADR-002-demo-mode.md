# ADR-002：体验模式用确定性脚本，不用 LLM

> 状态：已采纳 ｜ 2026-07-05

## 背景

在线 Demo 曾完全依赖真实引擎（agent-server + API key），导致 GitHub Pages 演示断链、获客演示不可用（PR #1 前的 P0 问题）。

## 决策

体验模式（`demoCoach.ts` + `demoBuilder.ts`）用**确定性脚本**驱动全部五阶段，制作阶段生成可真实操作的本地演示应用（Blob URL + localStorage）。作为静态托管环境的默认路径。

## 备选与否决理由

| 备选 | 否决理由 |
|---|---|
| 低价 LLM 跑演示 | 仍需密钥与后端，静态托管不可用；演示成本随流量线性增长 |
| 录屏视频代替 | 不可交互，说服力弱 |
| 免费额度 + 真引擎 | P3 账号体系之前无法防滥用 |

## 后果

- 正：零成本、零依赖、离线可演示；演示体验可控（不受 LLM 方差影响）
- 负：需与真实能力保持同步维护（能力边界文案、验收逻辑双份）；UI 明确标注"体验模式（脚本演示）"避免误导
