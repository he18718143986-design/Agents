# ADR-004：制作 Agent 只编辑声明式 modules.js，不自由写代码

> 状态：已采纳 ｜ 2026-07-05（PR #6 实现）

## 背景

L2 时期制作 Agent 自由生成单文件 HTML，质量方差大（布局/交互/中文处理不稳定），且无法安全对接数据底座。

## 决策

制作 Agent 的可编辑面收窄为：**`modules.js`（声明式模块定义：字段/汇总/预警规则）+ `custom.css`（CSS 变量）**。渲染引擎 `app.js`、认证、数据读写全部由模板固定实现，Agent 禁改（TEMPLATE_GUIDE.md 约束 + 提示词强调）。

## 备选与否决理由

| 备选 | 否决理由 |
|---|---|
| 自由编辑全部前端代码 | LLM 出错面大；改坏数据链路/认证的风险高；修改请求容易"推倒重来" |
| JSON 配置文件（非 JS） | 与现状差异小；JS 声明保留了受控的表达力（且可 `node --check` 验证） |
| 低代码平台（NocoBase）配置 | 见 ADR-003；自由度与品牌体验受限 |

## 后果

- 正：生成稳定性预期显著提升（待稳定性实验量化）；修改请求天然增量化；语法可机器验证
- 负：**模板引擎能力 = 生成应用能力上限**；超出声明式表达的需求（复杂联动、自定义视图）需扩展引擎或人工介入——这是有意的取舍，引擎能力随行业需求迭代扩展
- 衍生：quality 责任从"每次生成"转移到"模板引擎的一次性打磨"，符合平台化逻辑
