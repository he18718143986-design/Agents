# ADR-005：中国站默认模型 DeepSeek

> 状态：已采纳 ｜ 2026-07-05

## 背景

面向中国大陆公众提供生成式 AI 服务，不能直连境外 LLM（OpenAI/Anthropic 网络不可达且合规不允许）；同时单位经济要求算力成本低。

## 决策

中国站默认 **DeepSeek**（境内已备案、直连稳定、成本约低一个数量级）。模型作为**法域配置项**（环境变量 `LLM_MODEL`/`LLM_BASE_URL`），海外站（P4）切换 GPT/Claude/开源，核心代码不分叉。

## 备选与否决理由

| 备选 | 否决理由 |
|---|---|
| 通义千问/文心/智谱/Kimi | 均为可用备选（保留）；DeepSeek 编码能力与价格综合最优，且已完成全流程验证（#4） |
| OpenAI/Claude 经代理 | 违规服务境内公众 + 稳定性差，红线 |
| 自托管开源模型 | GPU 成本远超 API，P1–P3 不成立 |

## 后果

- 正：单次生成成本几毛–几元级，交付费定价空间大；已备案模型 API + 应用登记 = 较轻合规路径
- 负：供应商单点依赖；缓解：LiteLLM 层天然支持多供应商，备选模型清单保持可切换验证
