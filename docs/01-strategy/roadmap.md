## 阶段计划

### P1 内测（当前阶段）

**目标**：证明"能稳定交付真实可用的应用"，拿到首批付费验证。

| 事项 | 状态 |
|---|---|
| 稳定性实验（3 需求 × 2 次 + 修改请求，含 token 成本统计） | ⏳ 另一会话进行中 |
| baas-mvp 数据底座（黄金模板 + PocketBase） | ✅ 已合并 |
| 阿里云 ECS 部署（docker compose + Basic Auth，IP 访问） | ⏳ 待执行（部署包已就绪） |
| 5–10 个真实客户交付 + 三档定价访谈 | ⏳ 部署后开始 |
| ICP 备案（stagent.online） | ⏳ 用户推进中 |

**退出标准**：首次生成成功率 ≥80%、修改请求成功率 ≥70%、≥5 个真实交付、`c_create` 实测回填。

### P2 软上线（备案通过后）

- 域名 + HTTPS 上线，保持邀请制/Basic Auth
- 案例墙（已交付应用的公开展示，营销资产而非变现层）
- 托管订阅开始收费；每日备份、可用性监控
- 网站底部备案号、用户协议、隐私政策

### P3 平台化

- 账号体系 + 项目云端存储（替代 localStorage）（**v1 已提前落地**：平台 PocketBase 实例 + 属主隔离的 projects 集合，登录后云端同步、未登录本地回退，见 R10）
- 每制作任务独立沙箱（多租户隔离）
- 用量限额 + 入门额度漏斗；BYO key 分叉（此时才引入）
- 验收自动化：验收标准 → Midscene/Playwright 自动测试 + 证据报告（**v1 已提前落地**：确定性体检 + 截图证据，见 R9；Midscene 语义级验收待 P3 引入多模态模型后升级）
- 生成应用独立子域名托管（app-xxx.stagent.online，泛域名 + Coolify/Dokploy）

### P4 生态（生意 B 启动）

- 微信小程序通道（Taro 模板 + 第三方平台服务商代提审）
- 作品广场从案例墙升级为市场：模板分成、验证收入榜、买卖撮合
- 海外站（双栈架构，法域配置切换）

## 明确不做（当前有效的负面清单）

| 不做 | 原因 | 何时重新评估 |
|---|---|---|
| 原生安卓/iOS App | 签名/上架审核对非专业用户是硬门槛，小程序+PWA 覆盖大部分需求 | P4 后 |
| 期货/证券自动交易类需求 | 资金风险 + 合规，产品应识别并引导到合规路径 | 不重估 |
| 广告变现 | ARPU 与算力成本差 1–2 个数量级；引爆经营性 ICP/广告法负担 | P4 |
| 自建计费系统 | 支付意愿未验证前不投入 | P2 定价访谈后 |
| 完整生成式 AI 大模型备案 | 用已备案模型 API 走轻量登记路径 | 属地网信办另有要求时 |
| 通用 Agent 框架建设 | 用 OpenHands 挖深，不自研引擎 | 不重估 |
