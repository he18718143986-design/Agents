# 商业战略与单位经济

> 状态：**已采纳** ｜ 最后更新：2026-07-05 ｜ 决策 ADR：[ADR-007](adr/ADR-007-business-sequencing.md)

## 一句话定位

**让不会写代码的人，用自然语言 + AI，把想法做成能用、能分享的软件**——每个关键决策点用"人话"交还给用户拍板，其余交给 AI。

## 核心战略决策：两个生意，A 养 B

本项目存在两条商业轨道，**已决策按 A→B 排序，而非并行**（论证见 ADR-007）：

| | 生意 A：交付工具（先行） | 生意 B：创作者生态（后置） |
|---|---|---|
| 客户 | 中小商户老板（工程、教培、餐饮、美业等） | 创作者（创业者/产品/设计师/学生） |
| 产品 | "帮我做一个我业务要用的管理工具" | 作品广场、验证收入榜、买卖撮合 |
| 应用形态 | 私有工具，3–10 人用，无消费端流量 | 公开作品，需要陌生人浏览试用 |
| 收入 | 交付费 + 托管订阅 + 交付增值服务 | 市场分成 + 撮合佣金 + Pro +（可选）广告 |
| 启动条件 | 生成质量达标（进行中） | 平台上已有数百个活着的应用 |

**A 养 B 的逻辑**：交付生意积累的应用存量、行业模板、原生收款数据，正是生态生意的护城河原料（供给、验证收入、行业数据）。跳过 A 直接做 B = 双边市场冷启动，无现金流支撑。

## 目标客户（P1–P2 阶段）

主客户画像：**非技术背景的中小机构负责人**，特征：
- 有明确业务痛点（台账、预约、进度、课时），现用 Excel/纸质/微信群
- 外包报价 3–20 万且沟通痛苦；SaaS 不贴合自己的流程
- 不会也不想申请 API key、配服务器——**平台必须全托管**

代表案例（已验证需求）：工程部进度财务管控（需求一）、口才培训机构课时管理（需求三简化）、餐饮进销存、美容预约排班。

## 收入模型（按阶段启用）

| 阶段 | 收入线 | 定价假设（待内测访谈回填） |
|---|---|---|
| P1 内测 | 交付费（按项目） | 299 / 999 / 2999 三档试探支付意愿 |
| P2 软上线 | + 托管订阅 | 49–199 元/月（含数据备份、域名） |
| P3 平台化 | + Pro 订阅（自助生成额度） | 99–299 元/月 |
| P4 生态 | + 市场分成、撮合佣金、（可选）广告 | 参考应用商店 15–30%、撮合 3–5% |

**近期最强收入线是交付服务费**：AI 生成 + 关键节点人工兜底，对标 3–20 万外包市场，收几百到几千元仍有数量级性价比优势，且沉淀行业模板资产。

## 单位经济（骨架 + 回填状态）

核心原则：**贵的算力控制住，便宜的托管+收款留平台（沉淀验证收入护城河）。**

| 参数 | 含义 | 状态 |
|---|---|---|
| `c_create` | 单次创作会话 LLM 成本 | ✅ **实测（实验 001）：全流程上线 ~$0.02/次 ≈ ¥0.14**（仅生成 ~$0.011；异常上界 $0.047） |
| `G` | 体验到"哇"所需入门额度 | ⏳ P2 内测用 5 个真实客户测 |
| 交付费支付意愿 | 三档定价访谈 | ⏳ P1 内测收集 |
| 单应用托管成本 | PocketBase 实例 + 带宽存储 | 极低（SQLite 单文件，一台 ECS 可托管数百实例） |

**P1–P2 的成本结论（实测更新）**：平台持钥 + 单项目生成成本封顶。实测 `c_create ≈ ¥0.14/全流程`，即使按 10 轮迭代计算成本也在 ¥1.5 内；交付费按 ¥299 起 → **毛利率 > 99%**，无需 BYO 即闭环。**BYO key 漏斗推迟到 P3**（面向创作者人群时才有意义；中小商户老板不会配 key）。

## 法域与合规要点（中国先行）

- 引擎默认 **DeepSeek**（境内已备案模型，费用低、直连稳定）；不直连境外 LLM 服务境内公众
- 上线路径：ICP 备案（进行中，域名 stagent.online）→ 内测（IP + Basic Auth，不依赖备案）→ 备案通过后软上线；收费规模化前评估经营性 ICP 与生成式 AI 应用登记（用已备案模型 API 走登记路径，负担轻于完整备案，执行前咨询属地网信办确认）
- 许可证合规：技术栈全为 MIT/Apache（openhands-sdk、PocketBase、Express 等），未触碰任何 PolyForm 专有代码；上线前跑依赖许可扫描并生成 `THIRD_PARTY_NOTICES`
- 品牌："Stagent" 需完成中文名确定 + 商标检索（第 9/42 类），与 ICP 备案网站名称同步进行

## 北极星指标

**每周成功交付且 7 日内仍被真实使用的应用数**（定义与口径见 [指标定义](../02-product/metrics.md)）。

## 关键风险（P1–P2 视角）

| 风险 | 缓解 |
|---|---|
| 生成质量不稳定（当前最大） | 黄金模板收窄 LLM 出错面；稳定性实验设阈值门禁（首次成功 ≥80%、修改成功 ≥70%）才放内测 |
| 密钥/算力被盗用 | 内测期 Basic Auth；P3 前完成账号 + 限额 |
| 多用户并发互扰 | P3 前完成每制作任务独立沙箱 |
| 备案周期阻塞上线 | 内测走 IP 访问，不等备案 |
| 期望管理（"一句话出完美 App"） | 阶段门 + 能力边界显式告知（已实现） |
